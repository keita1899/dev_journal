# 要件定義

## 背景・目的

- 日報と技術記事を別々のアプリや、PC のフォルダで管理するのが面倒、かつわかりづらい
- 日報自体に学んだ事を書くと、日報の内容が長くなりぎる
- 技術記事によってわかりやすい文章の構造にしたい

## 想定ユーザー

- 現役エンジニア
- 初学者

## ユースケース

- メールアドレスとパスワードで新規登録・ログインできるようにしたい
- Github で新規登録・ログインできるようにしたい
- 日報を書きたい
- カレンダーで日報を書いた日がわかるようにしたい
- 技術記事を書きたい
- 内容に合わせて技術記事をわかりやすく書きたい
- 技術記事を書いたら、その日の日報に記録したい
- 技術記事に簡単に技術スタックを登録したい
- 技術記事を AI で要約したい
- 技術記事を Markdown 形式で出力したい
- 自分の技術スタックを登録したい
- Stripe でサブスクの決済ができるようにしたい

## 機能制限（プラン別）

### 無料プラン

- 技術記事タイプ制限あり
- 技術記事作成: 無制限
- 日報作成: 無制限
- Markdown 出力: 月 5 回まで
- AI 要約: 利用不可
- 技術スタック登録: 利用不可

### 有料プラン（Premium）

- 技術記事タイプ全て利用可能
- 技術記事作成: 無制限
- 日報作成: 無制限
- Markdown 出力: 無制限
- AI 要約: 月 100 回まで
- 技術スタック登録: 利用可能

## 非機能要件

### 基本要件

- モバイル対応（レスポンシブデザイン）
- サーバーは CloudRun
- Docker で開発
- バックエンドは RailsAPI
- フロントエンドは Next.js
- Github Flow

### セキュリティ要件

- SSL/TLS 通信の強制
- JWT 認証
- XSS 対策（入力値のサニタイズ）
- CSRF 対策（トークン使用）
- SQL インジェクション対策
- パスワードの暗号化（bcrypt）
- API レート制限

### パフォーマンス要件

- API レスポンスタイム: 95%が 500ms 以内
- ページロード時間: 3 秒以内
- 同時接続数: 1000 ユーザー
- 画像最適化（WebP 対応）

### 可用性要件

- 稼働率目標: 99.5%
- CloudRun の自動スケーリング
- ヘルスチェックエンドポイント
- エラー監視（Sentry 等）

### データ管理

- 日次バックアップ（Cloud SQL）
- バックアップ保持期間: 30 日
- 個人情報の適切な管理
- データ削除時の完全削除
